## hdLod Plugin ‚Äî WORKING IN USDVIEW üéâ

### Segfault Root Cause (found via GDB)
- `HdVisibilitySchema::Builder::Build()` calls `BuildRetained()` which is NOT thread-safe
- Storm queries `GetPrim()` from TBB worker threads concurrently
- Multiple threads calling `Builder::Build()` simultaneously corrupts shared state
- GDB: Thread 123 crashed at `HdVisibilitySchema::BuildRetained()`, null function pointer (0x0000000000000000)

### Fix
- Replaced `HdVisibilitySchema::Builder().SetVisibility(...).Build()` with `HdRetainedContainerDataSource::New()` directly
- Static singleton pattern for the invisible visibility data source ‚Äî thread-safe, no mutable shared state
- Also skipped xform reading for group position (`GetTypedValue(0.0f)` crashes on post-flattening xform data sources for non-renderable prims)

### Result
- UsdView runs without crashing
- High detail (Sphere) visible, Low detail (Cube) hidden via Hydra visibility overlay
- Camera at origin, threshold=15, active index=0 ‚Üí correct

### Lesson for Other Agents
**HdVisibilitySchema::Builder::Build() is NOT thread-safe in TBB contexts.**
Use `HdRetainedContainerDataSource::New()` directly when creating visibility data sources
that will be queried from TBB worker threads (which is always the case in Storm).

## hdLod Refactored with Proper Hydra Lifecycle

### Units' Key Insight (Lifecycle Table)
| Callback | Safe to read xforms? | Safe to write overlays? |
|----------|---------------------|-------------------------|
| _PrimsAdded | ‚ùå NO (constructing) | ‚úÖ cache only |
| _PrimsDirtied | ‚úÖ YES (valid when dirty) | ‚úÖ YES |
| GetPrim() | ‚ùå NO (must not read OTHER prims) | ‚úÖ overlay self only |

### Root Cause of Crashes
- matDs->GetTypedValue(0.0f) on flattened xform data source = null vtable
- Data source not fully initialized during _PrimsAdded scene population
- Even in _PrimsDirtied, our purpose-vis build's flattened xform crashes
- Workaround: read camera from USD stage (ComputeLocalToWorldTransform)

### Architecture (Final)
- _PrimsAdded: _RebuildGroupCache() + discover camera path (no xform reads)
- _PrimsDirtied: _UpdateCameraPosition() from USD stage + _EvaluateLod()
- GetPrim(): pure _InvisibleDataSource overlay (no side effects)
- _lodInitialized flag prevents eval before first camera dirty
